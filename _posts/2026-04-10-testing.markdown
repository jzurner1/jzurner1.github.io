
# Configurable files
- `cv.md` - base resume in markdown format
- `config/profile.yml` - personal details, target roles
- `modes/_profile.md` - more specific role context and personal policies
- `portals.yml` - keywords and searches
- `data/applications.md` - status of applications
- `data/pipeline.md` - list of considered job positions
- `data/scan-history.tsv` - history of job scans
- `reports/*` - detailed reports of various positions
- `output/*` - generated resumes

# Workflow

1. `/career-ops scan` to discover new open positions
2. `/career-ops pipeline` to look into those discovered positions
3. Tell it to generate CVs for the interesting positions
4. 

# Commands
### Scan for new job URLs (replaces /career-ops scan)
```
node scan.mjs              # Real run — writes to pipeline.md + scan-history.tsv
node scan.mjs --dry-run    # Preview only
node scan.mjs --verbose    # Show per-company job counts
node scan.mjs --level=2    # Greenhouse API only
npm run scan               # Same as node scan.mjs
npm run scan:dry           # Dry run + verbose
```

### Fetch job description
```
node fetch-js.mjs <url> <url>...
```

### Run full process on job site
```
/career-ops <job description>
```
